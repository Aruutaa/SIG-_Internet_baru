SDLC, database, dan analisis.

Bagian ini berisi uraian langsung untuk laporan pengembangan WebGIS fasilitas kesehatan.

Tujuan sistem SDLC Kelas user Database Analisis Pengujian

Tujuan sistem

GeoHealth Purworejo menyajikan data fasilitas kesehatan Kabupaten Purworejo dalam bentuk peta interaktif. Sistem memuat data rumah sakit, puskesmas, klinik, apotek, dan laboratorium. Informasi yang ditampilkan meliputi nama fasilitas, jenis fasilitas, alamat, kecamatan, layanan utama, status BPJS, jam operasional, kapasitas tempat tidur, jumlah dokter, jumlah tenaga kesehatan, serta koordinat lokasi.

Tujuan utama sistem adalah memudahkan pencarian fasilitas kesehatan, menampilkan sebaran fasilitas per wilayah, memperkirakan jangkauan layanan melalui radius, serta menyediakan halaman admin untuk pembaruan data.

SDLC yang digunakan

Model yang digunakan adalah Prototype. Model ini dipilih karena pengembangan WebGIS membutuhkan tampilan peta yang cepat diuji, mudah direvisi, dan dapat disesuaikan berdasarkan kebutuhan pengguna.

1

Perencanaan

Kebutuhan sistem ditentukan dari tujuan WebGIS, yaitu menampilkan data fasilitas kesehatan, menyediakan pencarian lokasi, memberi ringkasan analisis, dan menyiapkan halaman pengelolaan data.

2

Analisis

Data yang dibutuhkan terdiri atas atribut fasilitas kesehatan dan koordinat spasial. Pengguna dibagi menjadi pengunjung, admin, dan operator. Analisis yang dipakai meliputi sebaran fasilitas, radius layanan, fasilitas terdekat, dan komposisi fasilitas per kecamatan.

3

Desain

Desain sistem terdiri atas halaman beranda, peta, katalog data, login, admin, dan dokumentasi SDLC. Desain database menggunakan PostgreSQL/PostGIS agar data titik fasilitas dapat disimpan sebagai geometri.

4

Implementasi

Frontend dibuat dengan HTML, CSS, JavaScript, Leaflet, MarkerCluster, dan Chart.js. Backend disiapkan dengan PHP API. Database menggunakan PostgreSQL/PostGIS dengan tabel fasilitas, jenis fasilitas, layanan, relasi layanan, kunjungan bulanan, wilayah, pengguna, dan log aktivitas.

5

Pengujian

Pengujian dilakukan pada login, filter peta, popup marker, ekspor CSV, radius layanan, fasilitas terdekat, tambah data, edit data, hapus data, serta koneksi database.

6

Pemeliharaan

Pemeliharaan dilakukan melalui pembaruan data fasilitas, validasi koordinat, pengecekan layanan aktif, penyesuaian tampilan, dan backup database.

Kelas user

Pengunjung

Pengunjung dapat membuka peta, memakai pencarian, memfilter jenis fasilitas dan kecamatan, melihat popup atribut, menampilkan radius, melihat fasilitas terdekat, dan mengekspor data.

Admin

Admin dapat login, menambah data fasilitas, mengubah atribut, memperbaiki koordinat, menghapus data, dan mengelola informasi layanan.

Operator

Operator disiapkan sebagai role tambahan untuk validasi data lapangan sebelum data dipublikasikan.

Database geospasial

Database menggunakan PostgreSQL dengan ekstensi PostGIS. Data fasilitas disimpan dalam tabel fasilitas_kesehatan dengan kolom geometri bertipe Point, EPSG:4326. Struktur ini mendukung pemanggilan data ke peta, pencarian berdasarkan atribut, dan analisis jarak.

jenis_fasilitaskategori rumah sakit, puskesmas, klinik, apotek, laboratorium
fasilitas_kesehatanatribut utama, layanan, status, kapasitas, lat, lon, geom
layanan_kesehatanIGD, rawat jalan, rawat inap, KIA, imunisasi, farmasi, laboratorium
fasilitas_layananrelasi fasilitas dengan layanan yang tersedia
kunjungan_bulanandata jumlah kunjungan untuk grafik tren
app_users dan activity_logsakun pengguna dan catatan aktivitas admin

Analisis yang dipakai

Analisis sebaran fasilitas

Analisis ini menghitung jumlah fasilitas berdasarkan jenis dan kecamatan. Hasilnya digunakan pada kartu statistik, grafik komposisi, dan ringkasan wilayah.

Analisis radius layanan

Radius 1 km, 3 km, dan 5 km digunakan untuk memperkirakan area jangkauan dari satu fasilitas. Sistem menghitung fasilitas lain yang berada di dalam radius tersebut.

Analisis fasilitas terdekat

Sistem menghitung jarak lurus dari titik pengguna atau pusat Purworejo menuju fasilitas kesehatan. Hasilnya diurutkan dari jarak paling dekat.

Analisis atribut layanan

Status BPJS, IGD 24 jam, rawat inap, ambulans, jumlah dokter, dan kapasitas tempat tidur digunakan untuk membedakan karakter layanan tiap fasilitas.

Pengujian sistem

Pengujian dilakukan menggunakan skenario sederhana. Filter jenis fasilitas harus mengubah marker dan grafik. Filter kecamatan harus membatasi data sesuai wilayah. Tombol radius harus menampilkan lingkaran jangkauan. Tombol fasilitas terdekat harus menghasilkan urutan jarak. Form admin harus dapat menambah, mengubah, dan menghapus data. File SQL harus dapat dijalankan di pgAdmin tanpa error geometri Z.
