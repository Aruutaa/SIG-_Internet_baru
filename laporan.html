Rancangan dan tahapan pengembangan WebGIS.

Halaman ini disiapkan agar website tidak hanya menampilkan peta, tetapi juga menjawab poin tugas: SDLC, jenis user, dan basis data geospasial.

1. Software Development Lifecycle

Model SDLC yang dipilih adalah Prototype. Model ini digunakan karena kebutuhan WebGIS biasanya perlu diuji secara bertahap melalui tampilan peta, filter, simbol, dan form pengelolaan data. Dengan prototype, rancangan awal dapat langsung dicoba, kemudian diperbaiki berdasarkan evaluasi.

Analisis kebutuhan

Menentukan kebutuhan utama, yaitu peta fasilitas kesehatan, minimal dua jenis user, login admin, data spasial, dan database PostgreSQL/PostGIS.

Perancangan cepat

Menyusun struktur halaman, desain landing page, peta interaktif, katalog data, halaman login, dashboard admin, dan rancangan tabel database.

Pembuatan prototype

Mengembangkan tampilan website menggunakan HTML, CSS, JavaScript, Leaflet, serta API PHP sebagai penghubung ke PostgreSQL/PostGIS.

Evaluasi prototype

Menguji fitur filter, popup atribut, pencarian, statistik, rute, login admin, serta form tambah dan edit data.

Implementasi dan dokumentasi

Menyediakan file SQL, README, dan dokumentasi tahapan agar sistem dapat dijalankan kembali oleh pengguna lain.

Jenis user

Pengguna umum dapat membuka peta, mencari data, memfilter layer, membuat rute, dan melihat katalog data. Admin harus login untuk mengelola data fasilitas kesehatan. Role operator juga tersedia di database untuk pengembangan lanjutan.

Basis data

Database menggunakan PostgreSQL/PostGIS. Tabel utama yang digunakan adalah app_users, fasilitas_kesehatan, wilayah_administrasi, dan activity_logs. Kolom geometri menyimpan koordinat fasilitas dan batas wilayah.

Arsitektur sistem

Frontend terdiri dari HTML, CSS, JavaScript, Leaflet, Chart.js, dan Turf.js. Backend menggunakan PHP API. Database menggunakan PostgreSQL/PostGIS. Jika API belum aktif, peta tetap dapat berjalan melalui GeoJSON statis sebagai fallback.
