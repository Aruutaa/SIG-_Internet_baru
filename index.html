Tujuan sistem

Setiap fitur peta dan data disusun berdasarkan kebutuhan pengelolaan informasi fasilitas kesehatan.

01

Inventarisasi data

Data fasilitas, koordinat, alamat, jenis layanan, status operasional, BPJS, IGD, dokter, rawat jalan, rawat inap, dan konsultasi penyakit dalam.

02

Visualisasi spasial

Fasilitas ditampilkan sebagai marker, batas wilayah sebagai poligon, serta bangunan 3D dari OpenStreetMap sebagai konteks lingkungan.

03

Analisis akses layanan

Radius jangkauan, fasilitas terdekat, rute, dan filter layanan membantu membaca hubungan lokasi dengan kebutuhan pengguna.

04

Pengelolaan sistem

Halaman developer digunakan untuk login, buat akun, validasi data, pengeditan atribut, serta pemeliharaan struktur database.

Analisis utama

Peta tidak hanya menampilkan titik, tetapi juga membaca ketersediaan layanan.

1
Sebaran fasilitas per jenis dan kecamatan

Menunjukkan komposisi fasilitas kesehatan serta wilayah dengan data paling padat atau paling minim.

2
Filter layanan kesehatan

Data dapat disaring berdasarkan BPJS, IGD, rawat jalan/inap, ambulans, dokter, dan konsultasi penyakit dalam.

3
Radius jangkauan

Buffer 1 km, 3 km, dan 5 km dipakai untuk menghitung fasilitas yang berada dalam jangkauan titik tertentu.

4
Bangunan 3D OSM

Layer bangunan 3D digunakan sebagai konteks visual kawasan sekitar fasilitas kesehatan.
