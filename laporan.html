Laporan Sistem

SDLC, user, database, dan analisis.

Rancangan sistem memuat tahapan pengembangan, pembagian akses pengguna, basis data geospasial, dan analisis fasilitas kesehatan.

1. Software Development Lifecycle

Model SDLC yang digunakan adalah Prototype. Model ini sesuai karena WebGIS membutuhkan rancangan peta, filter, popup, simbol, radius, dan dashboard admin yang dapat diuji lebih awal lalu disempurnakan berdasarkan kebutuhan pengguna.

Perencanaan

Menentukan tujuan sistem, batas data, kebutuhan pengguna, jenis fasilitas kesehatan, atribut layanan, dan kebutuhan penyimpanan data spasial.

Analisis

Mengidentifikasi kebutuhan data titik fasilitas, batas wilayah, kelas user, alur login admin, serta analisis yang diperlukan: sebaran, radius layanan, fasilitas terdekat, dan atribut layanan kesehatan.

Desain

Menyusun struktur halaman beranda, peta, katalog data, login, admin, dan laporan. Database dirancang dengan PostgreSQL/PostGIS untuk menyimpan atribut dan geometri.

Implementasi

Frontend menggunakan HTML, CSS, JavaScript, Leaflet, MarkerCluster, Routing Machine, dan Chart.js. Backend menggunakan PHP API. Database menggunakan PostgreSQL/PostGIS.

Pengujian

Menguji filter fasilitas, pencarian, popup, radius, rute, ekspor CSV, login admin, form tambah data, edit data, hapus data, serta import SQL di pgAdmin.

Pemeliharaan

Memperbarui data fasilitas, memperbaiki koordinat, menambah atribut layanan, membackup database, dan menyesuaikan tampilan berdasarkan hasil evaluasi.

2. Tujuan sistem

Menginventarisasi fasilitas kesehatan, memvisualisasikan sebaran fasilitas, membantu pencarian fasilitas terdekat, menampilkan analisis radius layanan, dan menyediakan pengelolaan data oleh admin.

3. Kelas user

Pengunjung membuka peta, memakai filter, melihat popup, rute, radius, statistik, dan katalog data. Admin login untuk menambah, mengubah, menghapus, serta memvalidasi data. Operator disiapkan untuk membantu pembaruan data lapangan.

4. Database kesehatan

Database memuat app_users, fasilitas_kesehatan, wilayah_administrasi, jenis_fasilitas, layanan_kesehatan, fasilitas_layanan, kunjungan_bulanan, dan activity_logs. Tabel fasilitas menyimpan nama, jenis, kecamatan, alamat, telepon, jam layanan, BPJS, IGD 24 jam, rawat inap, ambulans, kapasitas tempat tidur, jumlah dokter, tenaga kesehatan, rating, dan geometri titik.

5. Analisis yang digunakan

Analisis meliputi sebaran fasilitas per jenis, sebaran per kecamatan, radius layanan 1 km sampai 5 km, fasilitas terdekat, rute menuju fasilitas, status BPJS, ketersediaan IGD, rawat inap, ambulans, dan ringkasan jumlah tenaga kesehatan.

6. Arsitektur sistem

Frontend menampilkan peta dan dashboard. Backend menghubungkan permintaan data dari browser ke database. PostgreSQL/PostGIS menyimpan atribut dan geometri. GeoJSON dipakai sebagai fallback saat API belum aktif.

Kesesuaian instruksi

Sistem memenuhi tiga poin utama tugas.

SDLC

Model Prototype dijelaskan per tahap dari perencanaan sampai pemeliharaan.

User

Tersedia pengunjung, admin, dan operator. Admin perlu login untuk mengelola data.

Database

Data atribut dan geometri disimpan dalam PostgreSQL/PostGIS serta dapat dibaca melalui API.
